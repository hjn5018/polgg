# 📋 Redis 라이선스 변경 및 보안 가이드 (FAQ)

본 문서는 2024년 Redis 라이선스 정책 변경에 따른 프로젝트 영향도와 소스코드 공개 의무에 대한 분석 내용을 담고 있습니다.

---

## 1. 개요: 무엇이 변했나요?
Redis는 2024년 3월부터 기존의 Open Source 라이선스(BSD)에서 **Source Available** 라이선스(RSALv2 및 SSPLv1)로 전환되었습니다. 
- **핵심 이유**: 아마존(AWS), 구글 등 대형 클라우드 사업자들이 Redis를 유료 서비스로 제공하면서 수익을 독점하고, 오픈소스 커뮤니티에는 기여하지 않는 문제를 해결하기 위함입니다.

---

## 2. 개발자 영향 분석 (Q&A)

### Q1. 이제 Redis를 쓰려면 돈을 내야 하나요?
**A. 아니요. 여전히 무료입니다.**
개인 사용자, 학생, 또는 일반 기업이 내부 인프라로 사용하거나 자사 애플리케이션(Ex: POL.GG)의 DB/캐시 용도로 사용하는 것은 라이선스 비용이 발생하지 않습니다.

### Q2. 제 프로젝트(POL.GG)의 소스코드를 강제로 공개해야 하나요?
**A. 전혀 그렇지 않습니다.**
라이선스에서 말하는 소스코드 공개 의무는 다음과 같은 특수 상황에만 발생합니다:
- **SSPLv1**: Redis를 '서비스 형태(Managed Service)'로 남에게 빌려주는 사업을 할 경우.
- **AGPLv3**: Redis 소스코드 자체를 수정하여 네트워크 서비스로 제공하는 경우.
- **단순 사용**: Redis에 데이터를 넣고 빼는 웹 애플리케이션 개발자는 소스코드 공개 의무가 **전혀 없습니다.**

### Q3. "유료화"가 걱정된다면 대안은 없나요?
라이선스 변경에 반발하여 탄생한 **완전한 오픈소스(BSD 라이선스 유지)** 프로젝트들이 있습니다:
- **Valkey**: 리눅스 재단 주도하에 AWS, Google 등이 참여하여 만든 Redis 포크입니다. 설정과 명령어가 Redis와 100% 호환됩니다.
- **KeyDB**: 고성능 멀티스레드 기반의 Redis 대체제입니다.

---

## 3. 결론 및 권장 사항

1.  **현상 유지 가능**: 현재 POL.GG 프로젝트에서 `redis:latest` 이미지를 사용하는 것은 법적으로나 비용적으로 아무런 문제가 없습니다.
2.  **보안/안정성 중시**: 만약 라이선스 이슈를 원천 차단하고 싶다면, `docker-compose.yml`에서 이미지 명만 `valkey/valkey:latest`로 교체하는 것을 추천합니다.
